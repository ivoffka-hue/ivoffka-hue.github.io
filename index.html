🗂 Структура файлов проекта

Система состоит из четырёх файлов и одной папки, которые должны лежать в одной директории:

📄index.html— главная страница с каталогом тестов (не редактируется)
📄test-engine.html— движок теста (не редактируется)
✏️tests-data.js— главный файл для редактирования: все тесты, вопросы, ответы
⚙️email-config.js— настройки email-рассылки
🖼images/— папка с картинками: logo.png и test-{id}.png для каждого теста
🛡 Главное правило №1 — резервная копия!
Перед любым изменением файлов сделайте архив всей папки проекта:
  1. Откройте папку с проектом в Проводнике
  2. Выделите все файлы и папки внутри (Ctrl+A)
  3. Правый клик → «Отправить» → «Сжатая ZIP-папка»
  4. Переименуйте архив, добавив дату: например, avz-testing-backup-2026-05-16.zip
  5. Сохраните в надёжное место (например, в облако или на отдельный диск)
Если что-то сломается после правок — распакуете архив поверх и всё вернётся как было.
Главное правило №2: вам нужно редактировать только файл tests-data.js. Все остальные файлы трогать не нужно.

📝 Чем открывать tests-data.js?

Это обычный текстовый файл. Откройте его любым из:

  • Notepad++ (бесплатно, рекомендуется) — меню «Файл» → «Открыть»
  • VS Code (бесплатно) — программа для разработчиков, с подсветкой
  • Обычный Блокнот Windows — работает, но неудобно на большом файле
⚠️ НЕ открывайте через Word или Excel — они испортят файл!

💾 Как сохранять изменения?

  • После правок нажмите Ctrl+S для сохранения
  • Затем обновите страницу браузера клавишей F5 — изменения применятся
  • Перед любым редактированием сделайте резервную копию файла (скопируйте рядом как tests-data-backup.js)

➕ Как добавить новый тест

Добавление теста состоит из двух обязательных шагов. Оба нужно сделать в файле tests-data.js.

Шаг 1
Добавьте карточку в раздел TESTS_META
Найдите в файле строку const TESTS_META = { — это начало каталога. Прокрутите до конца этого раздела (перед строкой с }; после блока всех тестов). Добавьте запись: 
"ваш-id": {
  title:      "Менеджер по закупкам",   // название вакансии
  category:   "Закупки",               // категория (придумайте сами)
  icon:       "📦",                    // эмодзи-иконка
  bannerColor:"#795548",               // цвет карточки (HEX)
  description:"Знание 44-ФЗ, работа с поставщиками, тендеры.",
  duration:   "20 мин",               // время на прохождение
  questions:  15,                     // число вопросов (всегда 15)
  salaryHint: "от 90 000 ₽",          // ориентир по зарплате
  status:     "active"                // active = доступен
},
💡 ID пишется латиницей без пробелов, через дефис: procurement-manager
Шаг 2
Добавьте вопросы в раздел TESTS_DATA
Найдите строку const TESTS_DATA = {. Прокрутите к концу раздела (перед строкой }; // конец TESTS_DATA). Добавьте: 
"ваш-id": {
  baseSalary:  90000,         // базовая ЗП для расчёта (Москва)
  levelScheme: "specialist",  // схема уровней (см. ниже)
  timeMinutes: 20,            // минуты на тест
  questions: [
    // 10 вопросов с ОДНИМ ответом:
    { type:"single", topic:"44-ФЗ",
      text:"Что регулирует 44-ФЗ?",
      options:["Закупки госорганов","Налоги","Трудовой кодекс","Таможню"],
      correct:[1],              // номер правильного варианта (с 0!)
      explanation:"44-ФЗ регулирует госзакупки."
    },
    // ... ещё 9 вопросов типа "single" ...

    // 5 вопросов с НЕСКОЛЬКИМИ ответами:
    { type:"multi", topic:"Поставщики",
      text:"Что проверяется при выборе поставщика?",
      options:["Цена","Репутация","Цвет логотипа","Сроки поставки","Наличие лицензий"],
      correct:[0,1,3,4],        // номера ВСЕХ правильных (с 0!)
      explanation:"Проверяют цену, репутацию, сроки и лицензии."
    },
    // ... ещё 4 вопроса типа "multi" ...
  ]
},
Схемы уровней (поле levelScheme):
  • "blue-collar" — рабочие специальности (грузчик, дворник, аппаратчик)
  • "specialist" — специалисты и инженеры (химик, технолог, валидатор)
  • "manager" — менеджеры и продавцы (KAM, маркетолог, ветконсультант)
  • "it" — IT-специалисты (разработчики, аналитики)
Шаг 3
Добавьте картинку (необязательно)
Положите файл test-ваш-id.png в папку images/.
Если картинки нет — отобразится эмодзи из поля icon.
Размер: рекомендуется от 600×200 пикселей, формат JPG или PNG.

✏️ Как изменить вопрос, ответ или обоснование

Откройте tests-data.js и найдите нужный тест по его ID (например, "driver"). Внутри — массив questions: [ ... ]. Каждый вопрос выглядит так:

{ type:"single", topic:"ПДД",
  text:"На каком расстоянии запрещена остановка у перекрёстка?",
  options:[
    "Ближе 3 м",   // вариант 0
    "Ближе 5 м",   // вариант 1  ← правильный
    "Ближе 10 м",  // вариант 2
    "Ближе 15 м"   // вариант 3
  ],
  correct:[1],
  explanation:"П. 12.4 ПДД: запрещено ближе 5 м от перекрёстка."
},

Что и как менять:

ПолеЧто этоПример изменения
textТекст вопросаИсправьте формулировку в кавычках
topicТема (бейдж)Напишите нужную тему: "Охрана труда"
optionsВарианты ответовИзмените текст в кавычках, сохраняя запятые
correctНомер(а) правильных ответовСчитайте с нуля: первый вариант = 0, второй = 1
explanationОбоснование ответаЗамените текст после завершения теста
⚠️ Важно про нумерацию: варианты считаются с нуля.
Если правильный — третий вариант в списке, то correct:[2] (не 3!)

Вопрос с несколькими правильными ответами:

{ type:"multi", topic:"Безопасность",
  text:"Что входит в предрейсовый осмотр?",
  options:[
    "Проверка масла",       // 0 ✓
    "Проверка шин",         // 1 ✓
    "Мойка автомобиля",     // 2 ✗
    "Проверка тормозов",    // 3 ✓
    "Медосмотр водителя"    // 4 ✓
  ],
  correct:[0,1,3,4],   // ← все правильные через запятую
  explanation:"Мойка не обязательна ежедневно."
},

Как добавить новый вопрос:

Найдите массив questions: [ нужного теста. Добавьте новый объект вопроса перед закрывающей ]. Не забудьте запятую после предыдущего вопроса:

  ...
  { type:"single", topic:"ПДД",
    text:"Предыдущий вопрос...",
    options:["А","Б","В","Г"],
    correct:[0],
    explanation:"..."
  },              ← запятая здесь обязательна!
  { type:"single", topic:"Новая тема",
    text:"Ваш новый вопрос?",
    options:["Вариант 1","Вариант 2","Вариант 3","Вариант 4"],
    correct:[0],
    explanation:"Объяснение правильного ответа."
  }               ← последний вопрос без запятой
]
💡 В каждом тесте должно быть ровно 15 вопросов: 10 с типом "single" и 5 с типом "multi".

🗑 Как удалить тест

Чтобы удалить тест, нужно убрать его из обоих разделов файла tests-data.js.

Шаг 1
Удалите карточку из TESTS_META
Найдите блок от "ваш-id": { до закрывающей }, этого объекта и удалите его целиком. 
  "driver": {              ← удалить с этой строки
    title: "Водитель...",
    category: "Логистика",
    ...
    status: "active"
  },                       ← до этой строки включительно
Шаг 2
Удалите вопросы из TESTS_DATA
Найдите блок "ваш-id": { в разделе TESTS_DATA и удалите его так же — от открывающей кавычки до закрывающей },.
💡 Альтернатива удалению: поставьте status: "draft" вместо "active" — тест исчезнет из каталога, но данные сохранятся. Это безопаснее.

Как скрыть тест (без удаления):

  "driver": {
    title: "Водитель персональный",
    ...
    status: "draft"   // ← было "active", стало "draft" — тест скрыт
  },

⚠️ Типичные ошибки и как их исправить

❌ После сохранения тесты пропали / страница пустая
Причина: синтаксическая ошибка в файле (пропущена запятая, лишняя скобка, незакрытая кавычка).
Решение:
  1. Нажмите F12 в браузере → вкладка «Консоль» (Console)
  2. Там будет красная ошибка с номером строки — перейдите к ней
  3. Или восстановите резервную копию файла
❌ Забыл поставить запятую между вопросами
{ ...вопрос 1... }   ← тут нужна запятая!
{ ...вопрос 2... }   ← иначе ошибка

// Правильно:
{ ...вопрос 1... },
{ ...вопрос 2... }
❌ Неправильно указал номер правильного ответа
Варианты нумеруются с нуля:
options: ["Вариант А", "Вариант Б", "Вариант В", "Вариант Г"]
//         ↑ 0          ↑ 1          ↑ 2          ↑ 3
correct: [2]   // правильный — "Вариант В"
❌ Новый тест добавил, но он не появился в каталоге
Проверьте:
  • ID в TESTS_META и TESTS_DATA должны совпадать точь-в-точь: "hr-manager"
  • Статус должен быть status: "active"
  • После запятой в конце предыдущей записи нет лишних символов
  • Обновите страницу браузера (F5 или Ctrl+R)
❌ В тексте вопроса нужны кавычки
Если текст содержит кавычки " — используйте одинарные кавычки для обёртки или экранируйте:
// Вариант 1 — одинарные снаружи:
text: 'Что означает термин "GMP"?',

// Вариант 2 — обратный слеш:
text: "Что означает термин \"GMP\"?",
Если ничего не помогает — напишите разработчикам: it@avz.ru и приложите скриншот ошибки из консоли браузера (F12).
